## hetzner2 Layout

### Pipeline directory: `/opt/pipeline/`

Git clone of this repo. Active working directory for pipeline runs.

```
/opt/pipeline/
  pipeline_core.py          — main entrypoint
  batch_runner.py           — Hetzner batch job launcher
  .venv/                    — Python 3.12 venv (all deps)
  logs/                     — per-run log files (neon_16s_*.log, etc.)
  receipts/                 — JSON receipt files from batch runs
  results/                  — analysis output CSVs, maps, reports
    bnf_trajectory_summary.csv
    ranked_candidates.csv
    taxa_enrichment.csv
    correlation_scan.json
    intervention_report.md / .json
    intervention_portfolio.csv
    spatial/
      bnf_spatial_map.png
    bnf_kriging_grid_conus.csv
    bnf_site_summary.csv
    keystone_organism_summary.csv
  models/                   — trained surrogate models
    bnf_surrogate_classifier.joblib
    bnf_surrogate_regressor.joblib
  FINDINGS.md               — auto-generated by findings_generator.py
```

**Note:** `/opt/pipeline` is behind the main git branch — it is updated by running `git pull` before each new run session. The `FINDINGS.md` and results here are the **latest generated outputs** even if code commits haven't been synced.

### Database: `/data/pipeline/db/soil_microbiome.db`

SQLite, WAL mode. ~948 MB. This is the **canonical data store** — distinct from the git repo.

```
/data/pipeline/
  db/
    soil_microbiome.db      — main DB (948 MB, WAL mode)
  receipts/                 — older receipt dir (Mar 7 batch)
```

See [database_schema.md](database_schema.md) for full table/column reference.

### API service

`uvicorn api.main:app` running on `127.0.0.1:8000`, 2 workers, managed by systemd as `pipeline-api`. Stays running continuously — this is not the pipeline runner, just an HTTP API over the DB.

```bash
# Check API status
ssh hetzner2 'systemctl status pipeline-api'

# View API logs
ssh hetzner2 'journalctl -u pipeline-api -n 50 --no-pager'
```

## Common Operations

### Check what's running
```bash
ssh hetzner2 'pgrep -a -f "pipeline_core|dfba|batch_runner" || echo "idle"'
ssh hetzner2 'uptime && free -h | grep Mem'
```

### Check DB state
```bash
ssh hetzner2 '
sqlite3 /data/pipeline/db/soil_microbiome.db "
SELECT
  (SELECT COUNT(*) FROM runs) AS total_runs,
  (SELECT COUNT(*) FROM runs WHERE t0_pass=1) AS t0,
  (SELECT COUNT(*) FROM runs WHERE t1_pass=1) AS t1,
  (SELECT COUNT(*) FROM runs WHERE t2_pass=1) AS t2;"
'
```

### Run activity by date
```bash
ssh hetzner2 'sqlite3 /data/pipeline/db/soil_microbiome.db "
SELECT DATE(run_date), COUNT(*), SUM(t1_pass), SUM(t2_pass)
FROM runs GROUP BY DATE(run_date) ORDER BY 1 DESC LIMIT 7;"'
```

### Tail live pipeline log
```bash
ssh hetzner2 'tail -f /opt/pipeline/logs/$(ls -t /opt/pipeline/logs/*.log | head -1)'
```

### Pull latest FINDINGS.md to local
```bash
scp hetzner2:/opt/pipeline/FINDINGS.md /home/seth/dev/soil_microbiome_core/FINDINGS.md
```

### Pull all results to local
```bash
rsync -avz hetzner2:/opt/pipeline/results/ /home/seth/dev/soil_microbiome_core/results/
```

### Sync code to hetzner2 (before a run)
```bash
ssh hetzner2 'cd /opt/pipeline && git pull'
```

### Start a pipeline run (T0 + T1 + T2)
```bash
ssh pipeline  # drops into /opt/pipeline with venv active
python pipeline_core.py --config configs/config.example.yaml -w 36 --fba-workers 18
```

## Git Sync State

The `/opt/pipeline` git checkout on hetzner2 is typically **behind** the main branch — it is updated manually before runs. Generated outputs (FINDINGS.md, results/) are **not committed** from hetzner2; they are pulled to dell5 and committed here.

Current divergence (as of 2026-03-10):
```
dell5 (local main):   2b5d343  ← authoritative
hetzner2 /opt/pipeline: bbf03a4  ← 14 commits behind
```

To sync hetzner2 before next run:
```bash
ssh hetzner2 'cd /opt/pipeline && git pull && pip install -r requirements.txt -q'
```

  ▼
